# AI Chat UI 升级计划

目标：把 `web` 的 Chat 对话页打磨成更接近主流 AI Chat 产品的体验，同时保持组件边界清晰、样式可维护、移动端可用。

## 当前状态

- Vue 迁移已完成，Chat 专用组件已集中到 `web/src/components/chat/`。
- 已完成消息气泡、Assistant 头像、思考块、工具调用卡片、工具详情侧栏、输入框基础体验升级。
- 已移除一批不必要的自维护组件，并接入 Element Plus。

## 执行顺序

1. [x] 流式输出体验
   - 靠近底部时自动跟随最新输出。
   - 用户手动上滑后停止强制滚动。
   - 显示“跳到底部”按钮。

2. [x] 消息状态体验
   - 发送中、失败、重试、停止后的状态更明确。
   - Assistant 空回复、异常回复有专门样式。
   - 用户消息支持发送时间或状态提示。

3. [x] 工具调用体验增强
   - 工具失败态更明显。
   - 工具卡片支持复制参数/结果。
   - 工具调用按阶段或类型分组展示。

4. [x] 思考过程体验增强
   - 当前运行步骤高亮。
   - 思考块摘要更明确，例如“完成 5 个步骤，调用 8 个工具”。
   - 失败或重试步骤有明显状态。

5. [x] Composer 输入区增强
   - 支持 Enter 发送 / Shift+Enter 换行的模式。
   - 支持粘贴文件或图片上传。
   - 附件上传增加进度、失败重试。

6. [x] 移动端体验
   - 底部输入区安全区适配。
   - 工具预览在手机上改成更轻的抽屉体验。
   - 长消息、代码块、表格触控体验继续优化。

7. [x] 可维护性收口
   - 评估 `PlanPanel` 是否迁入 `components/chat/`。
   - 评估 `AttachmentsMessage` 是否拆成 Chat 专用附件组件。
   - 为 Chat UI 增加统一导出入口。
   - 按需要继续拆分 `chat.css`。

## 验收规则

- 每完成一项都运行 `pnpm build`。
- 关键页面需要确认 `http://localhost:5173/` 可访问。
- 不引入新的全局样式重复项。
- 移动端宽度下不能出现横向溢出。
