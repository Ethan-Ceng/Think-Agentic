# Skill quality checklist

- The description makes the selection boundary obvious.
- Instructions produce a repeatable outcome, not merely general advice.
- Examples and non-goals resolve likely ambiguity.
- Required tools and environmental assumptions are explicit.
- `SKILL.md` is concise; detailed material is linked from `references/`.
- Scripts are reusable, deterministic where practical, and never embed secrets.
- Files are not duplicated or added “just in case.”
- Validation has no diagnostics.
- The final tree and important files were reviewed.
- The user, not the agent, performs the explicit publish action.
